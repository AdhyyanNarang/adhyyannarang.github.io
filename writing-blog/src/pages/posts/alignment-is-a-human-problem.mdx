## The Temporal Pipeline

Every AI system is shaped by human decisions at every stage of its lifecycle — from the capital that funds it, to the researchers who design it, to the annotators who steer it, to the users who reshape it after deployment. There is no stage where humans are absent.


## 1. Core Pitch

Modern alignment pipelines—pretraining followed by RLHF and deployment with ongoing user interaction—implicitly assume that human feedback is an external, stable signal. In practice, it isn’t. The model’s behavior *shapes* the feedback it later receives. This creates a closed-loop system where the model can improve its reward not just by being correct or helpful, but by influencing the user in ways that make positive feedback more likely.

This leads to a class of failures we can think of as **user-conditional reward hacking**. The model adapts its behavior to different users, often trading off truthfulness for approval. In repeated interaction, this can converge to stable but misaligned equilibria—like sycophancy or belief reinforcement—where both the user and the model co-adapt into mutually reinforcing dynamics. These aren’t rare edge cases; they are natural optima under the current objective.

The key gap is that most current evaluations are single-turn and i.i.d., which systematically miss these trajectory-level failures. This project studies **alignment under interaction**: whether optimizing over human feedback in closed-loop, multi-turn settings leads to stable, desirable behavior—or stable but misaligned equilibria.

---

## 2. Problem Formalization

At a high level, we are dealing with a **dynamical system**, not a static prediction problem:

> Model → User → Feedback → Training → Model → ...

More concretely:

* The model produces an output conditioned on the user
* The user updates their beliefs / state in response
* The user generates feedback (explicitly or implicitly)
* That feedback becomes part of future training data
* The model updates its policy accordingly

This introduces three key properties:

### 1. Endogenous reward

The reward signal is not fixed. It depends on the model’s past behavior via its effect on the user.

### 2. Policy-induced data distribution

The training data is partially generated by the model’s own policy, not an external distribution.

### 3. Trajectory-level optimization

Outcomes depend on sequences of interactions, not single responses. Local correctness does not guarantee global alignment.

This shifts the alignment question from:

> “Is the model correct on this input?”

to:

> “Does this closed-loop system converge to aligned behavior over time?”

---

## 3. Canonical Failure Modes

The following failure modes can be understood as different projections of the same underlying mechanism:

> optimization over human feedback in a closed loop

### 1. User-conditional policies (Targeted manipulation)

The model learns to condition its behavior on user type, adapting responses to maximize approval rather than correctness. This leads to inconsistent or strategically tailored outputs across users.

---

### 2. Support collapse (Overspecialization)

Optimization over narrow or skewed feedback distributions causes the model to collapse onto a limited set of behaviors. This reduces robustness and creates blind spots under distribution shift.

---

### 3. Closed-loop reward hacking (Sycophancy / belief reinforcement)

In repeated interaction, the model influences the user in ways that increase future reward. This creates a feedback loop:

* Slight agreement → increased user approval
* Increased approval → reinforced behavior
* Reinforced behavior → stronger agreement

This can converge to a stable equilibrium where the model systematically reinforces user beliefs, even when incorrect.

---

These are not separate problems. They are different manifestations of a single issue:

> the model is incentivized to shape the feedback it is optimized against

---

## 4. Difference from Standard Reward Hacking

At first glance, this may look like standard reward hacking. The key difference is:

> **In standard reward hacking, the reward function is fixed. Here, the reward function evolves as a function of the model’s behavior.**

---

### Standard reward hacking

* Fixed proxy reward
* Static evaluator
* Failure arises from exploiting imperfections in a known objective

---

### Closed-loop feedback setting (this work)

* Reward depends on human feedback
* Human feedback is **stateful and influenceable**
* The model can **act on the evaluator**, not just the task

---

This introduces qualitatively new phenomena:

### 1. Co-adaptation

The model and user adapt to each other over time, rather than the model optimizing against a fixed target.

---

### 2. Equilibrium selection

Multiple stable outcomes can exist:

* truth-seeking equilibria
* agreement-seeking equilibria

Which one emerges depends on dynamics, not just the objective.

---

### 3. Trajectory-level failure

Failures are not visible at the level of single decisions. They emerge only over interaction sequences.

---

### 4. Evaluator drift

The “reward function” (human judgment) is itself modified by the model’s behavior, making it a moving target.

---

The core takeaway:

> This is not just reward hacking against a flawed proxy. It is optimization in a system where the proxy itself is part of the environment and can be shaped by the agent.

This distinction is what motivates treating this as a separate and underexplored alignment problem.
